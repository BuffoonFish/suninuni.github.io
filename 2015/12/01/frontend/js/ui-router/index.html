Angular-Ui/ui-Router

任何一个MVC(MVVM)框架,无论是前端还是后端都有一套适合自身的路由规则.AngularJS原生的路由规则是ngRouter,使用起来极其方便.但是随着单页面应用的需求不断扩大, ngRouter已经不能满足很多情况,比如它不能实现多视图,不能实现视图的嵌套等等,缺点逐渐被暴露.ngRouter开发团队(准确的说是AngularJS开发团队)基于状态的方式重新实现了ngRouter,并将其从AngularJS中提取出写成了单独的模块,依赖于AngularJS,使用者可以根据需求安装ui-router.

ngRouter与ui-router的异同

  • ngRouter是基于url的改变驱动视图的变化，而ui-router是基于状态的改变驱动视图变化

  • 路由规则的配置的不同

  • ui-router可以实现同一页面多视图（views）和嵌套视图（parent）

state的配置

  • url：ui-router是基于状态的，所以会为每个状态设置一个url．
  • controller: controller会进行依赖注入，注入已存在的服务，或者resolve的返回值
  • resolve: 预注入服务或者数据,resolve需要和controller一起使用,如果resolve传入的是一个函数,那么这个函数的返回值会被解析成一个promise对象,并在controller实例化之前被注入到controller内,所以可以在controller内随意使用
  • params：可以在url中配置参数，这样在state被navigated或者transitioned的时候，可以用$stateParams拿到传递的参数．
  • views：多视图的时候使用．
  • onEnter / onExit：AngularJS会在进入视图和离开视图的时候调用这些回调函数，比如进入一个视图的时候弹出一个模态框，提示一些信息．
  • abstract(boolean)：如果被设置为true，意味着这个视图永远不能被直接active，只能当它的子状态被active的时候被间接active．
  • data：和resolve类似，但是它不会被注入到controller内，用于父状态向子状态传递数据，在子状态获取这些数据的常用方式是$state.current.data.(Issue: angular-ui-router源码中明确写到,data属性从父级的state继承而来,并且可以override这些data,反之是不可以的!见源码line 2191)

ui-router需要注意的问题

  • 如果子状态先注册，而子状态的父状态还没有注册，则子状态将会被缓存起来。但是父状态必须存在。
  • 如果子状态被激活，父状态会被隐式的激活。子状态将会把它的页面加载出来并嵌入到父状态的ui-view中

路由解析和模板渲染

  • 创建并存储一个state对象，然后调用$urlRouterProvider进行路由注册
  • AngularJS在digest循环开始时候会去监听$stateChangeSuccess事件，当state改变之后会去broadcast事件$stateChangeSuccess
  • 每次路由发生变化的时候会去在rule内去搜索，匹配到对应的路由规则
  • 通过调用$state.transitionTo()跳转激活对应的state(或者使用$state.go()进行状态间的跳转，并可以传递数据)，完成数据请求和模板的渲染
