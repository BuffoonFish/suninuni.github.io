那些SASS希望你知道的事

SASS(Syntactically Awesome Style Sheets)是一种CSS的预处理框架，它使编写CSS变得有组织性并富有逻辑，而不是一味平铺直诉，同样的也让大量的CSS变得易于维护。本文会解释一些SASS引以为傲的特性，这也是为什么我们会选择使用它的原因。

Variables

SASS允许声明以$开头的变量（通常情况下可以看做是常量，array与map类型的可以修改），而且变量名和值之间需要使用:分开，与CSS属性设置类似。

变量的作用域在{}内，如果该变量不在任何{}内，则为全局变量，在其他文件中也可以使用（这些文件是通过@import编译到一起的）。

建议所有的常量都使用变量，尽可能使用通用的全局变量（控制同一样式），这样在之后对样式的尺寸进行调整的时候仅需对变量进行修改。Bootstrap的个人定制就是通过对源码使用的全局变量进行修改，从而改变全局样式。

Nesting（嵌套）

嵌套分为选择器的嵌套和属性的嵌套，前者使样式的嵌套关系更加清晰，后者可以极大的减少代码量。

选择器嵌套

嵌套层次不应过深，最好小于3层，不大于5层。

属性嵌套

Mixins

Mixins类似与c++中的模板，可以将一部分样式抽出，作为单独定义的模块，被很多选择器重复使用。通过@mixin定义，@include或者+调用。SASS会将Mixins中的插入到调用此Mixins的选择器。Mixin可以定义参数与默认值，不同的选择器可以传入不同的参数，从而重用公用的CSS样式并设置特异化的属性。

但是这会造成编译出的CSS中有大量重复代码的冗余。
