Atom配置与插件

插件总是编辑器最重要的一部分，在这里记录一下我使用的一些比较好的插件，同时方便于自己在重新安装的时候不至于忘记这些优秀的插件。

至于为什么会选择Atom，因为它漂亮啊。

Packages

markdown-writer

markdown-writer: Make Atom a better Markdown editor and an easier static blogging tool.

既然是用markdown来写博客，对于我这种记性不太好的人来说，经常会记不清图片的引用格式或者link的引用格式等，这个插件熟练使用的话可以提高很多写博客的速度，包括博客模板等等。

Minimap系列

atom-minimap/minimap: A preview of the full source code.
atom-minimap/minimap-find-and-replace: Displays the search matches in the minimap.
atom-minimap/minimap-git-diff: A minimap binding for the Atom git-diff package.
jayk/minimap-autohide: Automatically hide the minimap when you aren’t using it.

用过sublime的都会记得其右边的那个对整个文件的‘缩略图’，其实单纯的Minimap的作用并不是很大，但如果搭配上以上配件的话，你就会发现这是一个极其方便的插件。

minimap-find-and-repleace 和 minimap-git-diff 是为了让你能更快的找到你想要找到的代码，我觉得这也是minimap存在的意义。前者标示出你所搜索的词语在文档中的位置，后者可以使你更快的移动到文档的相应位置，查看你在这次提交中所修改的代码。

minimap-autohide 是将minimap隐藏起来，只有当你滚动屏幕的时候，才会显示。我相信当你滚动屏幕的时候，你是想找到一些屏幕外的代码，而minimap的作用就是这，所以这个时候显示才会有意义。

autocomplete系列, lint系列, editorconfig

这些都是非常实用的插件，也是在不同的编辑器或者编译器中都是必不可少的。

file-icons

DanBrooker/file-icons: Assign file extension icons and colors for improved visual grepping.

既然选择Atom是因为它好看，那么你就不得不选择这个插件，多样的文件图标不仅能标示出当前文件的格式，更重要的是，它好看啊。

pigments

abe33/atom-pigments: A package to display colors in project and files.

这个插件可以让前端攻城师们清晰的知道自己所选择的颜色是什么，虽然通常情况下这个颜色是由UX定的，我们并不需要知道具体是什么颜色，但是，好看啊。

Summary

这篇文章还会时不时更新，如果遇到好用的插件，我都会记录在这里的。
